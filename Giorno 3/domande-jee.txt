37. Un Web Container è una parte del server che gestisce le applicazioni web Java. Si occupa di ricevere le richieste dal browser, farle elaborare al programma e inviare la risposta. Un esempio è Apache Tomcat.
38. Un sito web statico mostra sempre gli stessi contenuti a tutti gli utenti, perché è formato da pagine HTML già pronte. Un esempio è una semplice pagina personale in HTML.Un’applicazione web dinamica invece genera contenuti diversi in base all’utente o ai dati. Un esempio è Facebook.
39. Un file WAR è un file compresso che contiene un’applicazione web Java, con le sue classi, pagine HTML e librerie. È importante perché serve per distribuire facilmente l’applicazione su un server come Tomcat.
40. HTTP è il protocollo usato per la comunicazione tra browser e server. Quando un utente apre un sito, il browser manda una richiesta al server, il server la elabora e poi manda indietro una risposta con i dati da visualizzare.
41. Il file web.xml serve per configurare l’applicazione web. Permette per esempio di collegare un indirizzo URL a una Servlet, cioè dire quale parte del programma deve rispondere a una certa richiesta.
42. JEE (Java Enterprise Edition) e Jakarta EE indicano praticamente la stessa piattaforma per sviluppare applicazioni enterprise in Java. Il cambiamento è avvenuto quando Oracle ha trasferito la piattaforma alla Eclipse Foundation. Il nome è cambiato perché “Java” è un marchio di Oracle, quindi la nuova organizzazione non poteva continuare a usare “Java EE, ed è stato scelto il nome Jakarta EE.
